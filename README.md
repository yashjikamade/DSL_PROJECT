# ExpenseIQ — Setup & Run Guide

## Project Structure

```
expenseiq/
├── app.py                              ← Streamlit frontend (your file)
├── generate_dataset.py                 ← Backend: generates training data
├── train_model.py                      ← Backend: trains & saves the model
├── requirements.txt                    ← Python dependencies
├── monthly_spending_dataset_2020_2025.csv   ← Auto-generated by step 2
└── model.pkl                           ← Auto-generated by step 3 (optional)
```

---

## Quick Start (3 steps)

### Step 1 — Install dependencies
```bash
pip install -r requirements.txt
```

### Step 2 — Generate the dataset
```bash
python generate_dataset.py
```
This creates `monthly_spending_dataset_2020_2025.csv` (~60,000 rows of
realistic Indian household spending data from 2020–2025).

### Step 3 — (Optional) Pre-train and save the model
```bash
python train_model.py
```
Saves `model.pkl` and `model_metrics.json` with performance stats.
The Streamlit app trains inline anyway (cached), so this step is optional
but useful for checking model quality before running the UI.

### Step 4 — Run the app
```bash
streamlit run app.py
```
Open http://localhost:8501 in your browser.

## How the Backend Works

### Dataset (`generate_dataset.py`)
- Simulates 833 Indian households × 72 months (Jan 2020 – Dec 2025)
- Income spans ₹15,000 – ₹6,00,000/month across 4 tiers
- Realistic proportions for rent, groceries, transport, utilities, dining, entertainment, medical, savings
- Adds seasonal effects (festival spikes in Nov–Dec), yearly income growth (~5%), and natural noise

### Model (`train_model.py` / inline in `app.py`)
- **Algorithm**: Linear Regression (scikit-learn)
- **Features**: Income, Savings, Rent, Groceries, Transportation
- **Target**: Total Expenditure
- **Typical R²**: ~0.92–0.95 on test set
- **Typical MAE**: ~₹2,000–₹4,000

### Why Linear Regression?
- Fast to train, instant predictions
- Interpretable coefficients (each ₹1 more in rent → roughly ₹0.9 more total spend)
- Works well on structured, tabular financial data with strong linear correlations

## Customising the Model

To try a better model, swap in `app.py` / `train_model.py`:

```python
# Random Forest (more accurate, slower)
from sklearn.ensemble import RandomForestRegressor
model = RandomForestRegressor(n_estimators=100, random_state=42)

# Gradient Boosting (best accuracy)
from sklearn.ensemble import GradientBoostingRegressor
model = GradientBoostingRegressor(n_estimators=200, learning_rate=0.05)
```
